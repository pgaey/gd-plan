## 예시 (동네 미용실 예약)

## 한 줄 요약
동네 미용실을 1분 만에 예약하는 모바일 우선 사이트.
## 배경 / 맥락
전화 예약의 불편 + 정보 부족 해소. 디지털에 익숙한 환자 대상.
## 타깃 페르소나
- **단골 지수(30대)**: 니즈=빠른 재예약, 맥락=퇴근길 모바일
- **신규 방문자**: 니즈=시술/가격 비교 후 예약
## 핵심 가치
전화 없이, 빈 시간 확인부터 확정까지 한 흐름으로.
## 차별점 (선택)
'실시간 병원 관계자 확인'으로 자동예약 대비 신뢰/정확성.
## 사용자 역할 + 접근 모델 (roles)
- **roles**: [User(환자), Admin(병원 관계자)]
- **access model**: RBAC
- **디자인 함의**: Admin 전용 예약관리 화면 분리, User 는 본인 예약만 조회
## Capabilities
- `[CAP-01]` 시술 예약하기 (주체: User)
- `[CAP-02]` 예약 변경/취소하기 (주체: User)
- `[CAP-03]` 방문 후기 남기기 (주체: User)
- `[CAP-04]` 예약 확인/승인하기 (주체: Admin)
## 톤 키워드
친근, 깔끔, 신뢰
## 기기 / 플랫폼
Web (반응형, 모바일 우선)
## 성공 기준
1. 예약 완료까지 3탭 이내
2. 노쇼(No-show)율 감소
## 리스크 (선택)
낮은 온라인 예약 사용률(전화 선호) : 온보딩/혜택으로 유도
## Out of Scope
- 온라인 결제(현장 결제), 회원 등급제
