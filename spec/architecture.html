spec-13-01 · gd-plan 패키지 코어

상류 기획 레이어 아키텍처

지금 gen-design은 화면 하나하나를 /gd-chat으로 잘 만들지만, “이 프로젝트가 무엇이고·어떤 페이지가 있고·디자인 톤이 무엇인지”라는 전체 틀이 사람 머릿속에만 있습니다. gd-plan은 그 앞단에 기획 인터뷰 단계를 붙여, 문서 5종을 만들고 그것을 /gd-chat에 주입해 모든 화면을 일관되게 만듭니다.

한눈에 기획 문서 5종 핵심: /gd-chat 주입 일관성 게이트 패키지 구조 v1 / v2 범위 다음 단계

01한눈에 보기 — 전체 파이프라인

왼쪽에서 오른쪽으로 흐릅니다. 인터뷰(7 스킬) → 기획 문서(5종) → 화면 생성. gd-plan은 상류(새로 추가), /gd-chat부터 아래는 하류(기존 gen-design)입니다.

핵심 차별점 — 디자인을 0부터 생성하지 않고 66개 검증된 회사 디자인시스템 중에서 고릅니다. 그래서 결과가 들쭉날쭉하지 않습니다 (v0 / Lovable / Bolt 와의 차이).

02기획 문서 5종 — 무엇을 결정하나

인터뷰가 끝나면 docs/ 폴더에 5개 문서가 쌓입니다. 각 문서는 “한 가지 질문”에 답합니다.

prd.md

무엇을 만드나

타깃 페르소나·핵심 가치·사용자 capability·성공 기준·비범위.

design.md

어떤 톤·비주얼인가

collection에서 고른 디자인시스템 전체 + 고른 이유(reason). 사람이 쓰지 않고 AI가 추천, 이유만 사용자가 수정.

structure.md

어떤 페이지·구조인가

sitemap · 페이지별 섹션 · LNB/Sticky/Modal · 반응형 · empty/loading/error.

flows/*.md

사용자 여정은

auth·booking·admin 등 화면을 가로지르는 흐름. step마다 페이지·컴포넌트·데이터 명시.

ui-rules.md

수치 규칙은

섹션 간격(120/72px)·radius·모션(180ms)·CTA 위계 등 enforce 가능한 수치.

03핵심 — /gd-chat 주입 (F9)

이 spec의 존재 이유이자 가장 큰 리스크. 문서 5종을 “만드는 것”으로 끝나지 않고, /gd-chat이 화면을 만들 때 그 문서들을 읽어 반영해야 비로소 가치가 생깁니다.

방향은 한쪽입니다: 문서 → chat.md (단방향). chat.md를 바꿔도 문서는 바뀌지 않습니다. 그리고 만들기 전에 “이 화면이 structure.md에 정의돼 있나”를 검사해, 없으면 차단합니다.

04일관성 게이트 — /gd-plan-review

문서 5종이 서로 어긋나지 않게 잡아주는 단계. 핵심은 “요구사항 → 페이지 → 흐름”이 빠짐없이 연결됐는지 추적하는 것입니다.

예: PRD에 “방문 후기”가 있는데 어떤 페이지도 그것을 담당하지 않으면 차단(BLOCK). 이렇게 빠진 연결을 막아 일관성을 강제합니다.

v1
v1은 structural 누락 = BLOCK, 스타일/어휘 불일치 = WARN. 이 판정을 완전히 재현 가능한 방식(각 항목에 ID를 달아 집합 비교)으로 만드는 것은 v2로 미뤘습니다 — v1은 우선 동작 검증.

05패키지 · 배포 구조

새 패키지 @gen-design/plan은 기존 gd-skills 컨벤션을 그대로 따릅니다. 스킬의 “원본”은 create-gd-react preset이고, 나머지는 거기서 sync됩니다.

스킬 본문을 고칠 땐 preset의 .claude/commands/를 고치고, gd-skills/skills/는 빌드 시 자동 sync됩니다 (거꾸로 고치면 덮어써짐). _index.json은 매 호출마다 66개 파일을 다 읽지 않으려는 캐시입니다.

06이번 spec(v1) 범위 vs 나중(v2)

독립 검토(critique)를 거쳐, v1은 핵심 흐름 검증에 집중하고 견고성·정밀화는 v2 백로그로 기록만 해뒀습니다.

v1 — 지금 만든다 NOW

  • 7 스킬 + 5 템플릿 (기획 인터뷰)
  • _index.json 빌더 (collection 캐시)
  • /gd-chat 주입 (§1 / §5.10 / §6-8 / §12)
  • 일관성 게이트 — structural=BLOCK, style·vocabulary=WARN
  • create-gd-react 스캐폴드 (docs/ + 스킬 번들)
  • ADR 3개 (레이어별 SSOT / picker / 차단 정책)

v2 — 기록만 해둠 LATER

  • BLOCK 결정성 — ID + 집합 비교, gd-cli lint enforce
  • stale 감지 — _index.json 해시
  • staleness 전파 — 상위 문서가 더 새로우면 WARN
  • 인터뷰 중단·재개 상태 모델
  • 에러/부분상태 경로 (문서 일부만 있을 때)
  • design.md drift 감지 — sourceHash
